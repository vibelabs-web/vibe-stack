# PRD: 바이브랩스 커뮤니티 (Vibelabs Community)

> **MVP 캡슐**
> - **목표:** 초보자가 눈치 보지 않고 TIL을 기록하고 응원받는 따뜻한 정서적 유대감 형성
> - **페르소나:** 비전공자 독학 초보 '김코딩'
> - **핵심 기능:** 회원가입/로그인/로그아웃, 프로필 관리, TIL 게시판(글쓰기/조회/수정/삭제), 댓글/대댓글, 좋아요, 랭킹
> - **성공 지표:** 일일 활성 글 작성자 10명
> - **비기능:** 반응형 웹(PC/Mobile), ngrok을 이용한 로컬 서버 노출
> - **Out-of-scope:** 소셜 로그인, 알림 연동, 유료 결제, 전문적인 코드 리뷰
> - **리스크:** 사용자 참여 저조 (냉랭한 분위기)
> - **완화:** 운영자(김코딩)의 적극적인 선댓글 및 캐릭터 중심의 다정한 UI/UX
> - **실험:** MVP 출시 후 직접 7일간 매일 TIL 게시
> - **다음 단계:** 소셜 로그인 연동 및 실시간 알림 기능 도입

## 1. 문제 정의
"이런 사소한 걸 물어봐도 될까?" 
기존 개발자 커뮤니티는 너무 전문적인 용어와 차가운 분위기 때문에 초보자들이 질문하거나 자신의 배움을 드러내기에 심리적 문턱이 매우 높습니다. 이로 인해 초보자들은 고립감을 느끼고 학습 동기를 잃기 쉽습니다.

## 2. 목표 및 가설
- **목표:** 초보자들이 배운 내용을 마음껏 쏟아내고 '잘하고 있다'는 응원을 받을 수 있는 심리적 안전망 구축
- **가설:** 배운 것을 누군가에게 설명하듯 기록(TIL)하고 다정한 피드백을 받으면, 고립감이 해소되고 학습 지속성이 2배 이상 높아질 것이다.

## 3. 사용자 페르소나
### 김코딩 (29세, 비전공자 독학 초보)
- **배경:** 인문계 졸업 후 취업을 위해 코딩 독학 시작. 
- **니즈:** 오늘 배운 `for`문을 누군가에게 자랑하고 싶음. 에러가 났을 때 "괜찮아, 나도 그랬어"라는 공감을 얻고 싶음.
- **통점:** 스택 오버플로우나 대형 커뮤니티의 차가운 답변에 상처받은 경험 있음.

## 4. 사용자 스토리

### 인증 및 프로필
- **[회원가입]** 김코딩으로서, 이메일과 비밀번호로 간편하게 가입하여, 나만의 공간을 만들고 싶다. (**FEAT-AUTH-1**)
- **[로그인]** 김코딩으로서, 가입한 계정으로 로그인하여, 내 활동 기록을 이어가고 싶다. (**FEAT-AUTH-2**)
- **[로그아웃]** 김코딩으로서, 언제든 안전하게 로그아웃하여, 개인정보를 보호하고 싶다. (**FEAT-AUTH-3**)
- **[프로필]** 김코딩으로서, 나의 닉네임, 자기소개, 프로필 이미지를 설정하여, 커뮤니티에서 나를 표현하고 싶다. (**FEAT-PROFILE**)

### 게시판
- **[글쓰기]** 김코딩으로서, 오늘 배운 내용을 잊지 않기 위해, 코드 블록이 포함된 다정한 UI의 TIL 게시판에 글을 남기고 싶다. (**FEAT-POST-1**)
- **[글 조회]** 김코딩으로서, 다른 초보자들의 TIL을 읽으며, 나도 할 수 있다는 용기를 얻고 싶다. (**FEAT-POST-2**)
- **[글 수정/삭제]** 김코딩으로서, 내가 쓴 글을 수정하거나 삭제하여, 더 나은 기록을 남기고 싶다. (**FEAT-POST-3**)

### 댓글 및 대댓글
- **[댓글]** 김코딩으로서, 다른 친구의 글에 "정말 이해가 잘 돼요!" 같은 응원 댓글을 남겨, 함께 성장하고 싶다. (**FEAT-COMMENT-1**)
- **[대댓글]** 김코딩으로서, 댓글에 답글을 달아, 더 깊은 대화를 나누고 싶다. (**FEAT-COMMENT-2**)
- **[댓글 수정/삭제]** 김코딩으로서, 내가 쓴 댓글을 수정하거나 삭제할 수 있어야 한다. (**FEAT-COMMENT-3**)

### 좋아요 및 랭킹
- **[좋아요]** 김코딩으로서, 마음에 드는 글에 좋아요를 눌러, 응원의 마음을 전하고 싶다. (**FEAT-LIKE**)
- **[랭킹]** 김코딩으로서, 내 글이 받은 좋아요로 올라가는 랭킹을 보며, 성취감을 느끼고 내일도 공부할 힘을 얻고 싶다. (**FEAT-RANKING**)

## 5. 성공 지표
- **North Star Metric:** 일일 활성 글 작성자(DAU Writer) 10명
- **Input Metric:** 
    - 게시글 1개당 평균 응원 댓글 수 3개 이상
    - 주간 재방문율(Retention) 40% 이상

## 6. 리스크 보드
| 가정 | 리스크 | 완화 방안 | 트리거 | 대응 |
|---|---|---|---|---|
| 초보자들이 글을 쓸 것이다 | 아무도 글을 안 씀 | 초기 운영자의 적극적인 기록 | 3일간 게시글 0건 | 지인 초대 및 이벤트 |
| 댓글이 다정할 것이다 | 무플 또는 비난 댓글 | 신고 기능 및 다정한 톤 가이드라인 | 비난 댓글 1건 발생 | 즉시 삭제 및 경고 |
| ngrok 연결이 안정적일 것이다 | 접속 끊김 | 재연결 매뉴얼 제공 | 접속 불가 문의 3건 | 서버 상태 체크 가이드 배포 |
| 랭킹이 동기부여가 될 것이다 | 과도한 경쟁 심리 | '응원 많이 받은 순' 등 부드러운 순위 | 사용자 불만 접수 | 랭킹 기준 다각화 |
| 모바일 사용성이 좋을 것이다 | 레이아웃 깨짐 | 철저한 반응형 설계 | 모바일 폰트 깨짐 보고 | CSS 긴급 수정 |

## 7. Non-goals (범위 외 항목)
- 이메일 인증 (첫 버전은 간단한 이메일/비밀번호 가입만)
- 소셜 로그인 (구글, 카카오 등)
- 이미지 업로드 (첫 버전은 텍스트 및 코드 블록 집중)
- 실시간 채팅 기능
- 유료 구독 모델
- 비밀번호 찾기/재설정 (MVP 이후 도입)

## 8. 기능 상세 명세

### 8.1 회원가입 (FEAT-AUTH-1)
| 항목 | 내용 |
|------|------|
| 필수 입력 | 이메일, 비밀번호, 비밀번호 확인, 닉네임 |
| 이메일 검증 | 형식 검증 (`@` 포함), 중복 체크 |
| 비밀번호 규칙 | 최소 8자, 영문+숫자 조합 |
| 닉네임 규칙 | 2~20자, 중복 불가, 특수문자 제한 |
| 성공 시 | 자동 로그인 후 메인 페이지 이동 |
| 에러 처리 | 친절한 에러 메시지 표시 ("이미 사용 중인 이메일이에요") |

### 8.2 로그인 (FEAT-AUTH-2)
| 항목 | 내용 |
|------|------|
| 필수 입력 | 이메일, 비밀번호 |
| 인증 방식 | JWT 토큰 (Access Token + Refresh Token) |
| 토큰 저장 | HttpOnly Cookie 또는 localStorage |
| 성공 시 | 메인 페이지 이동, 헤더에 사용자 정보 표시 |
| 에러 처리 | "이메일 또는 비밀번호가 맞지 않아요" |

### 8.3 로그아웃 (FEAT-AUTH-3)
| 항목 | 내용 |
|------|------|
| 동작 | 토큰 삭제, 세션 무효화 |
| 성공 시 | 메인 페이지로 이동 (비로그인 상태) |

### 8.4 프로필 (FEAT-PROFILE)
| 항목 | 내용 |
|------|------|
| 조회 가능 정보 | 닉네임, 자기소개, 프로필 이미지 URL, 가입일, 작성 글 수, 받은 좋아요 수 |
| 수정 가능 정보 | 닉네임, 자기소개, 프로필 이미지 URL |
| 비밀번호 변경 | 현재 비밀번호 확인 후 새 비밀번호 설정 |
| 내 활동 | 내가 쓴 글 목록, 내가 좋아요한 글 목록 |

### 8.5 게시글 (FEAT-POST)
| 항목 | 내용 |
|------|------|
| 작성 | 제목(필수, 최대 100자), 본문(필수, 마크다운 지원), 코드 블록 |
| 조회 | 목록(페이지네이션, 최신순/인기순), 상세(조회수 증가) |
| 수정/삭제 | 본인 글만 가능, 삭제 시 soft delete |
| 권한 | 비로그인: 조회만 / 로그인: 작성, 수정, 삭제 |

### 8.6 댓글 및 대댓글 (FEAT-COMMENT)
| 항목 | 내용 |
|------|------|
| 댓글 | 게시글에 직접 작성, 최대 500자 |
| 대댓글 | 댓글에 대한 답글, 1단계 깊이만 지원 (무한 대댓글 X) |
| 수정/삭제 | 본인 댓글만 가능 |
| 표시 | 작성자 닉네임, 작성 시간, 내용 |

### 8.7 좋아요 (FEAT-LIKE)
| 항목 | 내용 |
|------|------|
| 대상 | 게시글 |
| 규칙 | 1인 1게시글 1좋아요, 토글 방식 (취소 가능) |
| 표시 | 게시글별 좋아요 수, 좋아요 여부 |

### 8.8 랭킹 (FEAT-RANKING)
| 항목 | 내용 |
|------|------|
| 기준 | 좋아요 수 기준 상위 게시글/사용자 |
| 종류 | 일간/주간/전체 랭킹 |
| 표시 | 순위, 닉네임, 좋아요 수, 대표 게시글 |
